# Report スキル

`/report [task-name]` コマンドに応答して、explore.md の調査結果を視覚的なレポートに変換するスキル。

## 目的

explore.md は調査の生データに近い。このスキルはそれを、テーブル・アスキー図・フローチャートなどを使って一目で構造が掴めるレポートに整形する。

## ワークフロー

### 1. explore.md を読む

`claude/tasks/[task-name]/explore.md` を読み込む。
存在しない場合は `/explore` を先に実行するよう伝えて終了。

spec.md があれば合わせて読み、元の調査意図を把握する。

### 2. 内容に合った図を選ぶ

調査内容を見て、最適な表現方法を判断する：

| 調査内容 | 表現方法 |
|---|---|
| ファイル間の依存・参照関係 | アスキー依存グラフ |
| 処理やデータの流れ | アスキーフローチャート |
| 設定値・API・パラメータの一覧 | テーブル |
| モジュール・レイヤー構成 | アスキーアーキテクチャ図 |
| 比較（ライブラリ、手法など） | 比較テーブル |
| 時系列の処理順序 | シーケンス図 |

一つのレポートに複数の図を組み合わせてよい。ただし図は意味があるときだけ使う。テキストで十分なら無理に図にしない。

### 3. report.md を生成する

`claude/tasks/[task-name]/report.md` に書き出す：

```markdown
# [task-name] レポート

## サマリー
調査で分かったことの要約（3〜5行）。

## [セクション名]

（テーブル、アスキー図、説明テキストを組み合わせて構成）

## [セクション名]
...

## キーポイント
- 重要な発見や注意点を箇条書き
```

### アスキー図のスタイル

```
┌──────────┐     ┌──────────┐
│ Module A │────▶│ Module B │
└──────────┘     └──────────┘
                       │
                       ▼
                 ┌──────────┐
                 │ Module C │
                 └──────────┘
```

- 罫線は `┌ ┐ └ ┘ ─ │ ▶ ▼` などの Box Drawing 文字を使う
- 矢印の向きでデータや依存の方向を示す
- ボックス幅は中のテキストに合わせる

### 4. 完了報告

レポートの構成（使った図の種類とセクション数）を簡潔にユーザーへ伝える。
