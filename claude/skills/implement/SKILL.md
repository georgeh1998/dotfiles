# Implement スキル

`/implement [task-name]` コマンドに応答して、`claude/tasks/plan.md` の実装計画をステップごとに実行するスキル。

## コマンド形式

- `/implement sample-feature` — 最初から（または続きから）
- `/implement sample-feature 2` — タスク2から開始

## ワークフロー

### 1. plan.md を読む

`claude/tasks/plan.md` を読み、実装タスクの一覧を把握する。存在しない場合は `/plan` を先に実行するよう伝えて終了。

### 2. 進捗を確認する

`claude/tasks/implement.md` が存在する場合：
- 開始番号の指定がなければ「Task [N] まで完了。Task [N+1] から再開しますか？」と確認
- 開始番号が指定されていればその番号から即開始

存在しない場合は新規作成：

```markdown
# [task-name] 実装ログ

開始: [日時]

## 進捗
[ ] Task 1: [タスク名]
[ ] Task 2: [タスク名]
...

---
```

### 3. タスクを順番に実行する

全タスクをユーザー確認なしで最後まで一気に実行する。各タスクについて以下を繰り返す：

#### A. タスクを提示する

```
─────────────────────────────────────
Task [N]: [タスク名]
─────────────────────────────────────
[概要]
```

#### B. 実装する

Edit/Write ツールで変更を実行する。完了後、変更したファイルを簡潔に報告してすぐ次のタスクへ進む。

#### C. 進捗を記録する

`claude/tasks/implement.md` を更新：

```markdown
## 進捗
[✓] Task 1: [タスク名]
[ ] Task 2: [タスク名]

### Task [N]: [タスク名] ✓
完了: [日時]

**変更内容:**
- `path/to/file.ext`: [変更の説明]

**詳細:**
[何をしたか・なぜしたか]

---
```

問題が発生した場合は `[!]` でマークし、手動対応の手順と再開コマンドを記録する。

### 4. 完了時

全タスク完了後、implement.md にサマリーを追記してユーザーに報告する。

## 注意事項

- 計画に書かれていない変更は加えない
- 問題が発生したら止まってユーザーに相談する
