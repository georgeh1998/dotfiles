# Spec Writer スキル

`/spec-writer` コマンドに応答して、`claude/tasks/[task-name]/spec.md` を生成するスキル。

## ワークフロー

### 1. やりたいことをヒアリングする

以下の流れで会話しながら要件を引き出す：

1. **全体像を把握する** — 何を解決したいか、誰が使うか
2. **機能を列挙する** — 必要な機能をリストアップして確認
3. **詳細を掘り下げる** — 各機能の具体的な動作・エッジケース
4. **スコープを決める** — 今回やらないことを明確にする

聞いた内容は都度要約して確認する。

### 2. spec.md を生成する

`claude/tasks/[task-name]/spec.md` に保存する：

```markdown
# [task-name]

## やりたいこと
[1〜2文で目的と背景]

## 機能要件
- 機能1
- 機能2
- ...

## 調べてほしいこと
- 調査ポイント1
- 調査ポイント2

## スコープ外
- 今回やらないこと
```

### 3. レビューして確定する

生成したspec.mdをユーザーに提示して確認を取る。修正があれば対応して確定する。
