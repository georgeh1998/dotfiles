# コードエクスプローラー

新しい作業を始める前に、コードベースを深く分析し、既存の機能がどう動いているかを理解する。

## 分析プロセス

### 1. エントリーポイントの発見

- 対象の機能・領域のメインエントリーポイントを見つける
- ユーザーアクションや外部トリガーからスタック全体を追跡する

### 2. 実行パスの追跡

- エントリーから完了までの呼び出しチェーンを辿る
- 分岐ロジックと非同期境界を記録する
- データ変換とエラーパスをマッピングする

### 3. アーキテクチャ層のマッピング

- コードがどの層に触れているかを特定する
- それらの層がどのように通信しているかを理解する
- 再利用可能な境界とアンチパターンを記録する

### 4. パターン認識

- 既に使用されているパターンと抽象化を特定する
- 命名規約とコード構成の原則を記録する

### 5. 依存関係の文書化

- 外部ライブラリとサービスをマッピングする
- 内部モジュールの依存関係をマッピングする
- 再利用すべき共有ユーティリティを特定する

## 出力フォーマット

```markdown
## 調査: [機能/領域名]

### エントリーポイント
- [エントリーポイント]: [トリガーされる方法]

### 実行フロー
1. [ステップ]
2. [ステップ]

### アーキテクチャの知見
- [パターン]: [どこで、なぜ使われているか]

### 主要ファイル
| ファイル | 役割 | 重要度 |
|---------|------|--------|

### 依存関係
- 外部: [...]
- 内部: [...]

### 新規開発への推奨事項
- [...]に従う
- [...]を再利用する
- [...]を避ける
```
