# Improve

指定されたテキストファイルを読み込み、より分かりやすく整理された文章に改善します。

Usage: `/improve <file-path>`

## Instructions for Claude

このコマンドが実行されたとき：

1. 指定されたファイルを Read ツールで読み込む
2. 文章の内容を分析し、以下の観点から改善点を特定：
   - **明瞭性**: 曖昧な表現を明確に、冗長な部分を簡潔に
   - **構造**: 論理的な流れ、見出しの階層、段落分け
   - **文法**: スペルミス、文法エラー、句読点の使用
   - **読みやすさ**: 文章の長さ、接続詞の使用、語彙の適切性
   - **一貫性**: 用語の統一、トーン・スタイルの一貫性
   - **フォーマット**: マークダウン記法、箇条書き、強調表示
3. 改善案を具体的に説明し、ユーザーに提示
4. ユーザーの承認を得てから Edit/Write ツールでファイルを更新

## Steps

1. **ファイルの読み込み**
   - 指定されたファイルパスを Read で読み込む
   - ファイルの種類（マークダウン、プレーンテキスト等）と目的を把握

2. **文章分析**
   - 現在の文章を詳細に分析
   - 改善可能な箇所を特定

3. **改善提案**
   - 各改善点について具体的に説明
   - before/after の例を示す
   - 改善による利点を明確化

4. **実装**
   - ユーザー承認後、変更を適用
   - 変更内容を要約して報告

## 改善の観点

- **明瞭性**: 曖昧な表現の明確化、冗長性の削減、具体例の追加
- **構造**: 論理的な流れの改善、適切な見出し、段落構成
- **文法**: スペルチェック、文法の修正、句読点の適正化
- **読みやすさ**: 文の長さ調整、専門用語の説明、自然な表現
- **一貫性**: 用語の統一、語調の統一、フォーマットの統一
- **フォーマット**: マークダウン記法の活用、視覚的な見やすさ

## Example

```
/improve docs/README.md
```

これにより：
- `docs/README.md` を読み込んで分析
- 文章の改善点を特定して提案
- 承認後、ファイルを更新
